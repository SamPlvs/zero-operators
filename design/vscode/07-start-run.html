Implementation notes

  • Panel is a VS Code WebviewPanel opened by the zo.startRun command — plain HTML controls, strict CSP, no libraries; submit posts form state to the extension host.
  • Defaults hydrate from .zo/config.yaml merged with plan frontmatter in .zo/plans/mnist-digit-classifier.md; mode preselects continue because STATE.md shows phases 1–3 complete.
  • Start run shells zo continue mnist-digit-classifier --repo ~/projects/mnist-delivery --headless-tmux --gate-mode supervised via a VS Code Task; the extension then watches .zo/runtime/session.json for the tmux session it now owns (zo-mnist-20260712-091412).
  • The command preview is the source of truth: form state serializes to argv verbatim, and the preview string is exactly what gets executed — --headless-tmux is always appended since the extension owns the session.
  • Preflight panel renders the check array from zo preflight --json one row per check; "Run preflight again" re-invokes it and diffs statuses in place. Warnings (e.g. mps-only GPU) never block; failed checks disable the primary button.
  • Bypass truth table: selecting full-auto forces the --bypass-permissions toggle on and locks it (flag omitted from argv — the CLI derives it); supervised and auto leave the toggle user-controlled.
  • --low-token maps to sonnet lead + haiku reviewers + capped iterations; the ~30% figure is the measured saving recorded in platform priors.
  • Deferred from v1: fuzzy plan picker across multiple workspaces and an editable delivery-repo path — v1 lists .zo/plans/*.md and reads the repo from plan frontmatter only.
