USER WORKFLOW

Five steps from repo to delivery
01
/project/import
›

Point ZO at any GitHub repo. It clones, analyzes the codebase (structure, dependencies, tests, README), detects the workflow mode, and drafts a complete plan.

Terminal
$ /project/import https://github.com/user/sensor-detector
Cloning into '../sensor-detector'...
Target file created: targets/sensor-detector.target.md
Memory initialized: memory/sensor-detector/
Analyzing codebase...
Detected: deep_learning mode (torch found)
Draft plan ready: plans/sensor-detector.md
8/8 plan sections complete. Review and approve.
02
Review plan.md
›

The plan has 8 required sections. Edit to sharpen objectives, set oracle thresholds, and add domain knowledge the agents missed.

plans/sensor-detector.md
1. Project Identity (name, version, status, owner)
2. Objective (specific deliverable)
3. Oracle Definition (primary metric, threshold, eval method)
4. Workflow Configuration (mode, gates)
5. Data Sources (files, APIs, loading code)
6. Domain Context (priors, constraints)
7. Agent Configuration (team composition)
8. Constraints (Python version, frameworks, limits)
03
/project/launch
›

Launch spawns the agent team. The Lead Orchestrator decomposes phases and assigns agents. Watch them work in tmux split panes.

tmux -- agent team

[lead-orchestrator] Phase 2: Feature Engineering
[data-engineer]    Feature pipeline constructed
[oracle-qa]        Gate 2: PASS -- awaiting human approval
04
Gate Review
›

In supervised mode, blocking gates pause for your review. You see metrics, decisions, and artifacts, then approve or reject with feedback.

COMMANDS

23 slash commands for Claude Code
› Project Lifecycle
/project/import <github-url>
Clone a repo, analyze the codebase, and draft a full plan.md
/project/connect <github-url>
Clone a repo and scaffold a ZO project target for it
/project/plan <task-description>
Create or update a structured task plan for the current project
/project/launch <project-name>
Launch an agent team to execute a project plan
› Memory & Continuity
/memory/recall <query>
Search project memory for decisions and priors matching a query
/memory/prime <project-name>
Load full project context -- the "catch me up" command for new sessions
/memory/priors <project-name>
Display all accumulated domain priors for a project
/memory/session-summary
Write a session summary and update all memory files
› Gate Management
/gates/approve
Approve the current pending gate and advance to the next phase
/gates/reject <reason>
Reject the current pending gate with a reason, triggering rework
/gates/gates <project-name>
Display all gates for a project with their current status
› Observability
/observe/watch <project-name>
Watch a running ZO session or show last known state
/observe/logs <project-name>
Display recent comms log events with optional type/agent filters
/observe/decisions <project-name>
Display all decisions from the project decision log
/observe/history <project-name>
Show a combined timeline of sessions, commits, and decisions
› Documentation

/document/validation-report <project-name>
Generate a comprehensive validation report from oracle results
› Agent Management
/agents/agents
List all defined agents with their roles, tiers, and teams

Spawn an agent in the current session with full context
/agents/create-agent <agent-name>
Create a new agent definition file interactively
› Utility
/commit
Stage and commit changes with a conventional commit message
