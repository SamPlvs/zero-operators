USER WORKFLOW

Five steps from repo to delivery
01
/project/import
›

Point ZO at any GitHub repo. It clones, analyzes the codebase (structure, dependencies, tests, README), detects the workflow mode, and drafts a complete plan.

Terminal
$ /project/import https://github.com/user/sensor-detector
Cloning into '../sensor-detector'...
Target file created: targets/sensor-detector.target.md
Memory initialized: memory/sensor-detector/
Analyzing codebase...
Detected: deep_learning mode (torch found)
Draft plan ready: plans/sensor-detector.md
8/8 plan sections complete. Review and approve.
02
Review plan.md
›

The plan has 8 required sections. Edit to sharpen objectives, set oracle thresholds, and add domain knowledge the agents missed.

plans/sensor-detector.md
1. Project Identity (name, version, status, owner)
2. Objective (specific deliverable)
3. Oracle Definition (primary metric, threshold, eval method)
4. Workflow Configuration (mode, gates)
5. Data Sources (files, APIs, loading code)
6. Domain Context (priors, constraints)
7. Agent Configuration (team composition)
8. Constraints (Python version, frameworks, limits)
03
/project/launch
›

Launch spawns the agent team. The Lead Orchestrator decomposes phases and assigns agents. Watch them work in tmux split panes.

tmux -- agent team

SAMPLE PROJECT

CIFAR-10 walkthrough -- test your ZO setup end-to-end
Quick Start

1. Setup & Initialize

./setup.sh                    # auto-fixes missing deps (uv, Claude CLI, zo)
zo init cifar10-demo          # scaffolds memory, targets, plans, delivery repo

2. Draft Plan

zo draft --project cifar10-demo -d "CIFAR-10 image classification, PyTorch CNN, 90%+ accuracy"
# Or interactively: zo draft --project cifar10-demo

Claude drafts the plan conversationally. Key fields:

Oracle
Test accuracy >= 0.85 (Tier 2)
Workflow
deep_learning mode, 6 phases
Agents
7 active (lead, data, model, oracle, review, test, research)
Gates
Gate 2 + Gate 5 blocking (human review)

3. Preflight + Launch

zo preflight plans/cifar10-demo.md    # validate environment
zo build plans/cifar10-demo.md        # launch agent team

4. Toggle Gates Mid-Session

# Start supervised, then switch once confident:
zo gates set auto --project cifar10-demo
zo gates set full-auto --project cifar10-demo   # full autonomy
What Happens
Phase 1 -- Data Pipeline
CIFAR-10 auto-download, DataLoaders with train/val/test splits, augmentation. Automated gate.
artifacts: data_quality_report.md, eda_summary.png, notebooks/phase/phase_1.ipynb
Phase 2 -- Input Representation
Normalization strategy, augmentation config (random crop, flip, color jitter). Blocking gate -- you review.
artifacts: feature_selection_report.md, notebooks/phase/phase_2.ipynb
Phase 3 -- Model Design
CNN architecture (ResNet-style or custom), loss function, optimizer + LR schedule. Automated gate.
artifacts: architecture_rationale.md, notebooks/phase/phase_3.ipynb
Phase 4 -- Training
Training inside Docker container (docker compose run --rm gpu python src/train.py). Iterates until oracle threshold met.
artifacts: training_report.md, training_curves.png, best_model.pt, notebooks/phase/phase_4.ipynb
Phase 5 -- Analysis
GradCAM visualizations, confusion matrix, per-class accuracy, ablation study. Blocking gate -- you review.
artifacts: analysis_report.md, confusion_matrix.png, notebooks/phase/phase_5.ipynb
Phase 6 -- Packaging
Inference script, model card, validation report, test suite. Automated gate.
artifacts: model_card.md, validation_report.md, inference.py, notebooks/phase/phase_6.ipynb
