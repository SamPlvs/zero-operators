shap/ log directorynew
Every prediction writes a SHAP vector to JSONL alongside the prediction value. Schema: timestamp, prediction_value, output_tag, top_10_features [{tag, shap_value, direction}], model_arm, regime. Enables the dashboard to render SHAP trending over time — not just static importance, but "which variables drove the last 24hr of predictions?"

Explainability agent — the ground truth question answered
The ground truth for explainability is three independent signals that must agree. No single signal is sufficient. The agent cross-references all three and produces a structured verdict. If two agree and one disagrees, it's QUESTIONABLE — escalate to domain evaluator. If physics check contradicts data corroboration, it's REJECT — something is wrong with the model.
Signal 1
Physics consistency
Ground truth: PHYSICS_PRIORS.md. Does this variable have the right causal relationship with this output according to epoxidation chemistry? Written by domain evaluator, read by XAI agent at every check.
Signal 2
Data corroboration
Ground truth: correlation_matrix.parquet from Phase 1. Does SHAP direction agree with what the historical data shows? Wrong sign = spurious relationship. Wrong magnitude = overfitting to noise.
Signal 3
Domain judgment
Ground truth: domain evaluator's process expertise. Handles cases the automated checks can't: novel relationships that are physically valid but historically rare, regime-specific behaviour, edge cases.
