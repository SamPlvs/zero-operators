## Decisions made

1. **Adaptations in plan, not agent files** — agent `.md` stays reusable; plan is single source of truth for project context
2. **Additive, not replacement** — appended to spawn prompt, preserves base instructions
3. **Plan Architect is the author** — adaptations emerge from scout findings, match the conversational draft flow
4. **Adaptations compose with custom agents** — orthogonal knobs, both available simultaneously, both work for core OR custom targets
5. **Two injection points** — inline in `_prompt_contracts` (so the Lead sees adaptation next to the contract for each phase's agents) AND top-level `# Per-project Agent Adaptations` section (so the Lead has an explicit summary and spawn-guidance block)
