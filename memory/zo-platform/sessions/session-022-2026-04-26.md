## ZO Platform Learnings

### Learning 1: Brand-redesign cascades extend to small assets

PR #51 swapped the entire palette / typography / mark, but the favicon, while small, was a forgotten cascade target. A 32×32 file is easy to miss when 11 files in `design/` and 14+ in `website/` are being replaced. PR-003 already names doc-cascade chains; the same principle applies to asset cascades — when the mark changes, every place the mark appears (favicon, logos, banner, app.js inline SVG, README references) needs an explicit checklist entry.

**Implication for ZO:** Future brand-identity changes should run a cascade audit step that greps for every small asset format (`.svg`, `.png`, `.ico`, inline SVG strings in JS) before declaring the redesign complete. Worth wiring into `validate-docs.sh` as an optional check that flags brand-identity files older than the most-recent `design/` mtime.

### Learning 2: SVG diagrams need measured boxes, not eyeballed boxes

The oracle text was 14px wider than its container, but the issue went unspotted from PR #51 → user-spotted in session 022. Rendered at the small scale used in the page, the overflow was subtle enough to miss visually. The fix used `getBBox()` to *measure* — that's the only reliable check for "does the text fit in this rect?".

**Implication for ZO:** When agents produce SVG diagrams (in reports, brand docs, etc.), the validation step should programmatically measure text bounding boxes against their container rects rather than relying on visual review. A simple "all-text-fits-its-box" assertion catches this class of error.

### Learning 3: Source-of-truth assets belong with their iteration toolchain

The brand-redesign-v2 PR shipped `readme-banner.{svg,png}` but not `workshop.png` or the render script. Future iteration would have required reverse-engineering. The fix here (consolidate everything into `design/banner/`) demonstrates a general principle: when the build step for an asset is non-trivial (composite, render, transform), the source files + render script + documentation belong with the output. A standalone PNG without its sources is an opaque artefact.

**Implication for ZO:** Project artefacts that require generation (plots, diagrams, reports) should ship alongside their generation script and inputs. The delivery repo's `experiments/exp-NNN/` pattern already does this for ML experiments; the same pattern applies to brand assets.

## Decisions Made

| ID | Decision | Rationale |
|----|----------|-----------|
| D-022a | Single-coral favicon over two-color (cream stroke + coral dot like `logo-dark.svg`) | Universal browser-tab visibility — cream fails on light tabs, ink fails on dark, coral works on both |
| D-022b | Flip hero column ratio to image-weighted, not increase aspect-ratio | Wider image with same shape reads as more weight; changing aspect-ratio just changes the shape |
