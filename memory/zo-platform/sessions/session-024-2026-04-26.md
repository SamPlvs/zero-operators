## What shipped

### PR #57 — `--low-token` mode core (merged)

Cost-saving preset for users on Anthropic Pro plans / student accounts / anyone watching their daily message budget.

- **CLI flag** `--low-token` and **plan field** `low_token: true` activate the same preset
- **Override flags** `--lead-model {opus,sonnet,haiku}`, `--max-iterations N`, `--no-headlines` compose with the preset
- **Precedence:** CLI flag > plan field > preset default > base default

The preset:

| Knob | Default | Low-token | Source |
|---|---|---|---|
| Lead model | opus | sonnet | ~5× cheaper rate |
| Phase-4 `max_iterations` | 10 | 2 | Cuts dominant Phase-4 multiplier |
| `stop_on_tier` | must_pass | could_pass | Earlier stop |
| `research-scout` cross-cutting | yes | no | Skip non-essential spawns |
| Haiku headline ticker | every 60s | disabled | ~60 calls/hr saved |
| Default gate mode | supervised | full-auto | No human-loop overhead |
| `CLAUDE_AUTOCOMPACT_PCT_OVERRIDE` env | unset | 60 | Earlier auto-compaction |

Implementation surface:

- `_LOW_TOKEN_PRESET` constant in [src/zo/cli.py](src/zo/cli.py:255) — single source of truth
- `LoopPolicy.low_token` + `_LOW_TOKEN_LOOP_CLAMPS` in [src/zo/experiment_loop.py](src/zo/experiment_loop.py)
- `Orchestrator.__init__(low_token=...)` threaded through `_agents_for_phase` (filters `research-scout`) and `build_lead_prompt` (drops dedicated adaptations section + compacts roster)
- `LifecycleWrapper.launch_lead_session(extra_env=...)` for env-var propagation (tmux: prefixed to command string; headless: `subprocess.Popen(env=...)`)
- New page [docs/concepts/low-token-mode.mdx](docs/concepts/low-token-mode.mdx) — preset table, FAQ, comparison, worked examples
- New page [docs/reference/low-token-preset.mdx](docs/reference/low-token-preset.mdx) — one-page reference card

### PR #58 — Doc fixes + sub-agent override (open, mergeable)

Three commits rolling up doc-bug fixes plus the post-bench architectural improvement:

**Commit `b453902`** — MDX `$` rendering bug + `/concepts/agents` redirect + bench findings

- Escaped 9 unescaped `$` occurrences in prose across `low-token-mode.mdx` (2), `quickstart.mdx` (1), `cost-benchmark.mdx` (6). Mintlify's MDX 2 parser was treating `$...$` as LaTeX math delimiters, mangling prose like "~$11 → ~$2-3"
- Permanent (301) redirect rule in `mint.json` for `/concepts/agents` → `/concepts/the-team` legacy slug
- Documented partial bench attempt findings in `cost-benchmark.mdx`

**Commit `1e5fad1`** — Sub-agent model override in lead prompt

- New `Orchestrator._prompt_low_token_overrides()` method
- When `low_token=True`, lead prompt gains a "Low-Token Sub-Agent Model Override" section instructing the Lead Orchestrator to pass `model="claude-sonnet-4-6"` to every `Agent()` spawn
- Bypasses Claude Code 2.1.92's TeamCreate-spawns-Opus-regardless-of-frontmatter behaviour
- Includes graceful fallback clause if `Agent()` ignores the parameter
- 2 new tests verify section presence in low-token mode and absence in default mode (706 total)

**Commit `10c492c`** — `the-team` 404 root cause fix

- Replaced `files <500 lines, functions <50 lines` with `files under 500 lines, functions under 50 lines` on `docs/concepts/the-team.mdx:51`
- **Root cause of the persistent 404:** MDX 2 parser treats `<` followed by any non-whitespace as a JSX opening tag, so `<500` was a malformed JSX attempt → page failed to compile → Mintlify silently dropped it from the build
- Diagnosed via direct HTTP probe — sibling concept pages return 200, `/concepts/the-team` returns 404
- Repo-wide regex sweep with code-block exclusion confirmed this was the only such pattern
- Bonus: stale `docs.zero-operators.dev` URL corrected to `docs.zerooperators.com` (actual deploy) across README badge / docs/README / STATE.md
