## Bugs encountered + resolved

### Bug 1: Pytest-MPS tensor extraction returns garbage values
CIFAR-10 oracle tests failed with `IndexError: index 13806002416 is out of bounds for axis 0 with size 10` despite the same `evaluate()` working during training and in standalone scripts. Three fix attempts (`.tolist()` → `.cpu().tolist()` → `torch.bincount`) all produced the same garbage values under pytest. Root cause: PyTorch 2.x MPS regression in pytest's process model — even after `.cpu()`, tensor reads can return uninitialised memory.

**Resolution:** Pin test-time `evaluate()` to `torch.device("cpu")`. 10K test set is fast on CPU; training stays on MPS via `select_device("auto")`. Documented as PR-034 in PRIORS.md.

### Bug 2: Existing test broken by platform-aware scaffold
`test_cli.py::TestScaffoldDelivery::test_scaffold_creates_compose` was hardcoded to assert `"capabilities: [gpu]"` in the emitted compose. After auto-detection, on a Mac dev box this correctly flips to CPU template. Updated test to pass `gpu_enabled=True` explicitly so it's host-independent.
