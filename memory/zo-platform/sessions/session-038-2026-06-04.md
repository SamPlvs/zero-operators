## Objective

Let a user run a second ZO session (`zo report`) on the same project as a live
`zo continue` model session — for report verification + LaTeX writing — without
overloading one context window or racing on shared state. Designed
interactively with the user: **1 model + 1 report**, surrogate-write +
consolidate, auto-on-last-close + manual `zo consolidate`,
artifacts-live / memory-snapshot, worktree-branch artifacts, **Opus** report
writing (never Haiku).

## Shipped

- **`src/zo/surrogate.py`** — git worktree on `report/<id>` + per-PID liveness
  lock files + delta-memory store at `.zo/surrogates/<id>/`.
- **`src/zo/consolidate.py`** — fold surrogate deltas into canonical (append
  decisions, dedup priors, copy summaries; **never** STATE), commit + merge the
  report branch when no peer is live, archive. Idempotent fold-marker prevents
  double-fold across deferred merges.
- **`src/zo/report.py`** — Opus report-lead prompt: verify via `oracle-qa` +
  `data-engineer`, write LaTeX to the worktree, hard read-only boundaries on
  canonical memory + experiments.
- **`memory.py`** — `flock` on `append_decision` / `append_prior`.
- **CLI** — `zo report` (`--objective` / `--resume` / `--no-tmux` /
  `--bypass-permissions` / `--no-consolidate`) + `zo consolidate` (`--dry-run`);
  `_launch_and_monitor` gains per-PID registration + auto-consolidation on
  last-close + role-gated overlay cleanup.

## Safety hardenings (PRIORS PR-045)

1. A report session **never touches the shared permission overlay** (cleanup
   gated to orchestrator-role) — can't disrupt a running `zo continue`, even one
   predating the liveness registry.
2. `--no-consolidate` suppresses exit-merge for that predates-the-registry case.
3. `apply_bypass_overlay` **inherits** an already-active bypass overlay (no-op)
   instead of clobbering it — closes a lingering-bypass bug (model exits first →
   repo left in bypass mode).
